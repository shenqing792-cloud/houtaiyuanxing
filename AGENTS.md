# 项目说明

## 项目定位

这是一个用于需求评审演示的后台原型项目。用户是产品经理，目标是在本地快速搭建可操作的管理后台页面，用于向开发演示页面结构、筛选项、表格、按钮状态和基础交互。

当前项目不是完整前端工程，只有一个静态页面：

- `index.html`：发行运营后台 / 风控管理 / 平台封禁页面。

后续扩展时，优先继续在现有静态 HTML/CSS/JS 结构上迭代，除非用户明确要求改成 Vue、React 或真实 Ant Design 项目。

## 视觉与风格

页面目标是贴近现有管理后台和 Ant Design 风格，而不是做营销页或展示页。

整体风格要求：

- 后台工具型界面，信息密度较高。
- 使用蓝色作为主色，深蓝顶部导航和侧边栏。
- 内容区以白底卡片、浅灰边框、浅蓝表头为主。
- 字号偏小，主体表格和筛选项多为 `12px` 或 `13px`。
- 按钮、输入框、下拉框高度多为 `32px`。
- 表格线、输入框边框使用浅灰蓝色。
不要把后台页面做成大卡片堆叠、营销式 hero 或过度装饰风格。

## 当前页面结构

`index.html` 当前包含这些主要区域：

- 顶部系统导航：发行运营后台、悟道、青木、曙光等。
- 左侧菜单：运营管理、转端管理、直播管理、风控管理等。
- 面包屑与历史页签区域。
- 主内容卡片 `section.panel`。
- 子页签按钮：
  - `封禁操作`
  - `操作记录查询`
- 筛选区。
- 表格工具栏。
- 表格。
- 分页。

子页签不是两个独立按钮，而是同一组紧密排列的切换页签。有多少个子页签，就放多少个按钮。

## 筛选项规则

当前筛选区使用 CSS Grid：

- `.filter-grid`：默认 4 列。
- `.filter-row`：内部为标题和控件两列，标题右对齐。
- 标题靠上对齐，不强制垂直居中，尤其是 textarea 类型筛选项。
- 筛选项标题和控件整体应右对齐贴近参考图。

展开/收起规则：

- 展开时展示当前页签的全部筛选项。
- 收起时只展示当前页签下的前三个可见筛选项，并保持一行。
- 收起按钮文案：
  - 展开状态：`∧ 收起`
  - 收起状态：`∨ 展开`

当前已有筛选项类型：

- 日期范围：自定义日期范围选择器。
- 单时间：日期 + 时分秒，自定义日期时间选择器。
- 单选下拉：自定义 `.select-control`。
- 输入框：分为不可拉动短输入框、可拉动多行输入框。
- 范围输入：左右两个输入值，中间用短横线分隔。
- 多选下拉：自定义 `.multi-control`，非必填，支持全选、全不选、反选。
- 操作按钮区：查询、重置、展开/收起。

组件沉淀页面：

- `components.html`：组件沉淀统一入口，左侧只保留真实存在的页面标签，并在同一个地址内切换内容。

组件素材页：

- `filters.html`：用于记录和展示筛选项组件效果，后续新增筛选项类型优先沉淀到这个页面。
- `list-components.html`：用于记录和展示列表单元格内容形态，后续新增列表内容类型优先沉淀到这个页面。
- `modal-components.html`：用于记录和展示添加/编辑窗口组件形态，包括居中弹窗、右侧抽屉和表单校验。
- 这些素材页支持 `?embed=1` 嵌入模式，被 `components.html` 加载时会隐藏自身顶部栏和左侧栏。

## 日期范围组件

日期范围组件不是简单样式，应具备可操作能力：

- 显示格式：`YYYY-MM-DD - YYYY-MM-DD`。
- 不包含时分秒。
- 点击打开日期面板。
- 日期面板左侧有快捷选择：
  - 昨日
  - 今日
  - 本月
  - 最近7天
  - 最近30天
  - 本季度
  - 本年
- 右侧展示双月日历。
- 支持选择开始日期和结束日期形成范围。

后续增加日期组件时，应复用当前结构和交互，不要只画一个静态框。

## 单时间组件

单时间组件用于选择一个具体时间点。

结构与交互规则：

- 显示格式：`YYYY-MM-DD HH:mm:ss`。
- 包含日期、时、分、秒。
- 点击打开日期时间面板。
- 面板顶部展示日期输入框和时间输入框。
- 日期区展示月历，可切换月份并选择具体日期。
- 时间区展示三列：时、分、秒，可分别选择。
- 底部有 `此刻` 和 `确定`。
- `此刻` 会填入当前日期时间，`确定` 关闭面板。

## 单选下拉组件

下拉组件使用自定义结构 `.select-control`，不要直接使用原生裸 `select` 作为主要展示。

结构要点：

- `.select-trigger` 作为点击区域。
- `.select-text` 展示当前值或 placeholder。
- `.select-menu` 展示选项。
- 每个选项必须带 `.select-option` 类，否则样式和点击逻辑不会生效。

必填与非必填规则：

- 筛选项里一般不使用红色星号表示必填。
- 必填字段通过默认选中第一个值体现。
- 必填字段不可清除。
- 非必填字段默认展示灰色 placeholder。
- 非必填字段选中后，鼠标悬停框体时，下拉箭头变成清除按钮，可点击清空。

当前示例：

- `所属项目`：必填，默认选中 `青木`。
- `操作对象`：非必填，placeholder 为 `请选择操作对象`。
- `操作类型`：仅在操作记录查询页签展示，非必填，选项为 `封禁`、`解封`。

## 输入框组件

输入框分两类：

- 不可拉动：用于短文本，如 `AppId`。高度通常为 `32px`，不允许拖动改变高度。
- 可拉动：用于批量输入，如平台账号 ID、设备号、IP。使用 textarea，右下角可拖动，仅扩展高度。

## 范围输入组件

范围输入组件用于填写一个数值或金额区间。

结构与交互规则：

- 一个控件内包含左右两个输入框。
- 左侧通常为最小值，右侧通常为最大值。
- 中间使用 `-` 分隔。
- 两侧输入框共享一个外层边框，整体高度通常为 `32px`。
- 示例：`角色总充值`，placeholder 为 `最小角色总充值`、`最大角色总充值`。

## 多选下拉组件

多选下拉当前只沉淀非必填形态。

结构与交互规则：

- 使用自定义 `.multi-control`。
- 默认展示灰色 placeholder。
- 下拉面板顶部有快捷操作：`全选`、`全不选`、`反选`。
- 选项左侧为文本，右侧用对勾表示选中。
- 已选择内容直接显示在框体内。
- 框体内仅展示第一个选中的值，后续选中数量以 `+数字` 展示。
- 选择后，鼠标移动到框体右侧下拉箭头区域时显示 `X`，点击清除所有选择。

## 表格规则

当前表格是自定义 HTML table：

- 使用 `table-layout: fixed`。
- 使用 `colgroup` 控制列宽。
- 表头浅蓝底。
- 表格行有浅灰蓝边框。
- 文本过长时用省略号。
- 表格横向溢出时允许横向滚动。
- 表格底部有分页。

列宽规则：

- 每个表格类型有默认列宽。
- 每个表格类型有最小列宽。
- 默认宽度会根据当前表格容器宽度自动补齐。
- 补齐宽度优先分配给长文本列，如操作内容、详情、操作原因、操作时间。
- 不要把多余宽度堆到最后一列后面形成空白。
- 拖动列宽时，只修改当前列宽，不联动改变其他列宽。
- 刷新页面后恢复默认列宽，不做本地持久化。

操作列规则：

- 操作列要预留足够宽度，保证两个按钮同一行展示。
- 当前操作列默认宽度约 `200px`，最小宽度约 `190px`。

## 列表内容组件

列表单元格内容主要沉淀以下形态：

- `列表内容总览` 这一行应尽量集中展示文本、可点击样式、状态图标、开关、直接编辑样式，避免为基础样式拆出过多重复示例块。
- 文本显示：最常规的纯文本展示。
- 状态图标：用于状态、入口开关状态等展示，不允许在列表中直接修改，使用明显的标签或状态图标表示当前状态。
- 开关：可在列表中直接切换状态；切换前必须弹出二次确认弹窗，用户确认后才改变状态。
- 可点击样式：蓝色文本，并携带信息图标，点击后打开新窗口或详情页。
- 直接编辑样式：蓝色文本，并携带编辑图标，点击后打开对应操作窗口。
- 操作项：
  - 简单操作只有编辑、删除等，使用纯图标按钮。
  - 复杂操作使用文字按钮。
  - 操作项较多时，将低频操作收纳到 `更多操作` 下拉菜单中。

列表图标规则：

- 列表内容页中的图标统一按 Element Plus Icon 风格处理。
- 编辑图标使用 Element Plus 的 `EditPen` 图标。
- 删除图标使用 Element Plus 风格的垃圾桶图标。
- 信息、下拉等图标也保持同一套 Element Plus 图标视觉，不混用字符图标。

批量编辑规则：

- 批量编辑有两种常见入口，均沉淀在 `list-components.html` 的列表内容组件页。
- 带选择列的批量编辑：
  - 表格第一列为复选框。
  - 表头复选框支持全选/全不选。
  - 未选中数据时，批量修改按钮为禁用态。
  - 选中数据后，批量修改按钮变为亮色可点击。
  - 批量修改按钮旁展示 `已选 N 条`。
  - 选中行使用浅蓝底色突出。
- 不带选择列的批量编辑：
  - 列表左上角直接展示亮色批量修改按钮。
  - 点击按钮后打开批量修改窗口，在窗口中选择项目并填写修改内容。

## 添加/编辑窗口组件

添加/编辑窗口主要沉淀在 `modal-components.html`。

窗口样式有两类：

- 页面居中弹窗：适用于内容较少、操作聚焦的新增/编辑场景。
- 页面右侧抽屉：适用于内容较多、希望保留页面上下文的新增/编辑场景。
- `窗口样式（添加）` 用于展示新增场景，窗口关闭后再次打开应恢复初始空表单状态。
- `窗口样式（编辑）` 用于展示编辑场景，允许存在不可编辑字段。

两类窗口都应包含：

- 弹窗标题。
- 右上角关闭按钮。
- 底部取消按钮和确认按钮。
- 窗口配置内容。

窗口配置内容规则：

- 单选下拉框：
  - 必填时标题前显示红色 `*`。
  - 交互与筛选项组件的单选下拉一致。
- 输入框：
  - 必填时标题前显示红色 `*`。
  - 分为不可拉动短输入框和可拉动多行输入框。
  - 有字数限制时，在框体右下角显示字数统计，例如 `0/200`。
- 范围输入组件：
  - 必填时标题前显示红色 `*`。
  - 结构与筛选项组件的范围输入一致。
- 单选项：
  - 必选。
  - 必须有默认值。

必填校验规则：

- 点击确认按钮时触发表单校验。
- 必填项未填写时，对应控件边框变红。
- 控件下方显示红色错误文案。
- 校验通过后再执行提交或关闭窗口。

编辑态禁用规则：

- 编辑窗口中某些字段可被人为限制不可编辑。
- 禁用态控件置灰，鼠标悬停显示禁止操作光标。
- 单选下拉框、输入框、范围输入组件、单选项都需要支持禁用态。

开关确认弹窗规则：

- 标题为 `提示`。
- 左侧使用黄色感叹号提示图标。
- 文案说明本次操作影响，例如 `此操作将关闭闪验一键登录，是否继续？`。
- 底部按钮为 `取消` 和主按钮 `确定`。

## JS 组织方式

当前页面使用原生 JavaScript。

关键结构：

- `rows`：封禁操作表格数据。
- `logRows`：操作记录查询表格数据。
- `tableProfiles`：不同子页签的表头、列宽、最小列宽、补齐列、数据源配置。
- `setActiveTab(tab)`：切换子页签。
- `renderColumns()`：渲染 colgroup 和表头。
- `renderTable()`：按当前页签渲染表格。
- `installColumnResizing()`：安装列宽拖拽。
- `updateVisibleFilterRows()`：处理筛选区展开/收起。

后续新增表格类型时，优先新增或扩展 `tableProfiles`，避免复制整张表格。
